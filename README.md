# AuthDRF — кастомная система аутентификации авторизации
## Описание ролей и прав доступа
---
В качестве бизнес модели я решил выбрать личный блог одного автора с несколькими уровнями подписки.
**Роли**:
- Неаутентифицированный гость. Может просматривать ресурсы не требующие аутентификации:
	- `/auth/sign-up`
	- `/auth/sign-in`
- Аутентифицированный гость. Всё то же что и неаутентифицированный гость. Может просматривать главную страницу блога, свои личные данные, бесплатный контент, а также имеет возможность разлогиниться:
	- `/my`
	- `/my/password`
    - `/`
	- `/public/<id>` (страницы с бесплатным контентом)
- Платный подписчик. Всё то же, что и аутентифицированный гость. Может просматривать контент для подписчиков:
	- `/sub/<id>`
- Премиальный подписчик. Всё то же, что и платный подписчик. Может просматривать контент для премиальных подписчиков:
	- `/premium/<id>`
- Админ. Всё то же что и платный подписчик. Имеет доступ к админ панели
	- `/admin/users`
	- `/admin/roles`
	- `/admin/permissions`
